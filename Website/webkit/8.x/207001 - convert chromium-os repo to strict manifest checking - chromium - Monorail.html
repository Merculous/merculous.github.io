wipe local copies where portage-stable renders it unneeded

Identified via:
targets=$(for x in $(pquery --raw --repo chromiumos -n 2> /dev/null); \
  do [ "$(pquery --raw --repo portage-stable $x 2> /dev/null| wc -l)" != 0 ] \
  && echo $x; done)
{ pquery --attr repo --repo chromiumos ${targets} --raw; \
  pquery --attr repo --repo portage-stable ${targets} --raw; } | sort -g | \
  less

Finally, note that in some cases we're wiping a *higher* version than
is in portage-stable.  Frequently people pulled things into portage-stable
than never keyworded it, or induced breakage and never cleaned up the left
overs.  Thus since I need to bring the cpv count down, they're getting
cleaned if they can be (vast majority have sat for multiple months).

Purpose of this is to bring the ebuild count down minimizing how much
manifest data we have to worry about integrating in, plus the cleanup
is rather needed going by the size of this list.
